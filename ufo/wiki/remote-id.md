# Remote ID (14 CFR part 89)

[[faa|FAA]] rule (codified at 14 CFR part 89) that requires most drones operated in U.S. airspace to broadcast a real-time identification and position signal. Often described as a "digital license plate" for drones. Compliance has been phased in since late 2023.

## Relevance to UAP / counter-UAS vault

- **Federal baseline for facility-side detection.** Under the FAA's proposed [[unmanned-aircraft-flight-restriction|UAFR]] rule (May 6, 2026, [[src-faa-drone-rule-critical-infrastructure-2026-05]]), §74.56 makes **Remote ID receiving capability** the federal minimum standard a critical-infrastructure operator must already have in place before applying for a UAFR.
- Operators may deploy receivers in-house or contract to third-party vendors (e.g., [[airsight]] AirGuard).
- Establishes the sensing layer below the [[counter-uas]] kinetic/electronic-defeat layer — Remote ID does *not* grant interception or jamming authority, only situational awareness.
- Cooperative protocol: relies on compliant drones broadcasting their own identity, so does not cover hostile/non-compliant operators (the actual incursion threat in the [[drone-incursions|2023-2026 incursion pattern]]).
